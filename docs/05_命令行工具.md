# 命令行工具

安装 Drun 后，你将获得 `drun` 命令行工具。

## 命令概览

```bash
$ drun --help

Commands:
  run / r          运行测试用例
  q                快速调试（无需 YAML）
  init             初始化测试项目
  convert          格式转换（cURL/HAR/Postman）
  convert-openapi  OpenAPI 转换
  export curl      导出为 cURL
  tags             列出标签
  check            语法检查
  fix              自动修复
  server / s       报告服务器
```

## drun q（快速调试）

快速调试模式。无需 YAML 和环境文件，直接执行 HTTP 请求（httpie 风格）。

### 基本用法

```bash
# 简单 GET 请求
drun q https://httpbin.org/get

# POST JSON 请求
drun q https://httpbin.org/post -X POST -d '{"name": "test"}'

# 带请求头
drun q https://api.example.com/users -H 'Authorization: Bearer token123'

# 带 Query 参数
drun q https://httpbin.org/get -p foo=bar -p page=1
```

### 常用参数

```bash
drun q <url> [OPTIONS]

必需参数:
  url                   请求 URL（必须以 http:// 或 https:// 开头）

可选参数:
  -X METHOD            HTTP 方法（默认: GET）
  -H HEADER            请求头（可多次），格式: 'Key: Value'
  -p PARAM             Query 参数（可多次），格式: k=v
  -d DATA              请求体字符串；支持 -d @file.json 读取文件
  -validate EXPR       断言表达式（可多次）
  -extract EXPR        提取变量（可多次），格式: name=$expr
  -max-body CHARS      终端输出 body 最大字符数（默认: 2048）
  -o FILE              保存完整响应体到文件
  -save-yaml FILE      转存为 YAML 用例
  -v                   输出请求/响应头等详细信息
  -secrets plain      显示敏感字段明文（默认）
  -secrets mask        脱敏敏感信息
```

### 断言与提取

```bash
# 状态码断言
drun q https://httpbin.org/get -validate 'eq status_code 200'

# JSON 响应断言（JMESPath）
drun q https://httpbin.org/get?foo=bar -validate 'eq $.args.foo bar'

# 响应头断言
drun q https://httpbin.org/get -validate 'contains headers.Content-Type json'

# 提取变量
drun q https://httpbin.org/get?id=123 -extract 'myId=$.args.id'

# 组合使用
drun q https://api.example.com/users \
  -validate 'eq status_code 200' \
  -validate 'gt $.total 0' \
  -extract 'firstId=$.data[0].id'
```

支持的断言操作符：`eq`、`ne`、`contains`、`not_contains`、`regex`、`lt`、`le`、`gt`、`ge`、`len_eq`、`len_gt`、`len_ge`、`len_lt`、`len_le`、`in`、`not_in`、`contains_all`、`match_regex_all`、`exists`。

### 转存为 YAML

```bash
# 将请求转存为 YAML 用例
drun q https://httpbin.org/post -X POST -d '{"name": "test"}' \
  -validate 'eq status_code 200' \
  -save-yaml testcases/from_q.yaml

# 脱敏后转存
drun q https://api.example.com/login -X POST \
  -H 'Authorization: Bearer secret123' \
  -d '{"password": "secret"}' \
  -save-yaml testcases/login.yaml -secrets mask
```

### 退出码

- **0**: 请求成功，所有断言通过
- **1**: 断言失败
- **2**: 参数错误或请求失败

### 请求体文件写法

```bash
# 创建请求体文件
echo {"name": "test"} > body.json

# 使用文件传递请求体
drun q https://httpbin.org/post -X POST -d @body.json
```

## drun run（运行测试）

运行测试用例，`r` 是 `run` 的简写。

### 基本用法

```bash
# 运行单个测试（支持省略 .yaml 后缀）
drun run test_login -env dev
drun run testcases/test_login.yaml -env dev

# 运行目录下所有测试
drun run testcases -env dev

# 运行测试套件
drun run testsuite_smoke -env dev
```

### 常用参数

```bash
drun run <path> [OPTIONS]

环境参数:
  -env NAME            环境名称，优先加载 .env.<name> 文件
  -env-file FILE       显式指定环境文件路径
  # 如果当前目录存在 .env，未传 -env 时会自动加载

可选参数:
  -k TAG_EXPR           标签过滤表达式
  -vars k=v            覆盖变量（可多次使用）
  -html FILE           生成 HTML 报告（临时单文件运行默认不生成）
  -report FILE         生成 JSON 报告
  -allure-results DIR  生成 Allure 结果
  -failfast            首次失败时停止
  -secrets mask        脱敏敏感信息
  -secrets plain      显示敏感信息
  -response-headers    记录响应头
  -log-level LEVEL     日志级别（DEBUG/INFO/WARNING/ERROR）
  -log-file FILE       日志文件（临时单文件运行默认在当前目录生成一个日志文件）
  -notify CHANNELS     通知渠道（feishu,dingtalk,email）
  -notify-only POLICY  通知策略（always/failed/passed）
  -snippet off          禁用代码片段生成（临时单文件运行默认已关闭）
  -snippet MODE   片段模式（off/all/curl/python）
```

### 示例

```bash
# 带 HTML 报告
drun run testcases -env dev -html reports/report.html

# 单文件自动加载当前目录 .env
drun run test_tts.yaml

# 非脚手架的临时单文件运行默认只生成一个日志文件
drun run ./test_tts.yaml -env-file ./demo.env

# 显式环境文件
drun run test_tts.yaml -env-file ./demo.env

# 标签过滤
drun run testcases -env dev -k "smoke and not slow"

# 覆盖变量
drun run test_login -env dev -vars username=admin -vars password=123456

# 调试模式
drun run test_login -env dev -log-level DEBUG -secrets plain

# 快速失败
drun run testcases -env dev -failfast

# 发送通知
drun run testcases -env dev -notify feishu -notify-only failed
```

## drun init（初始化项目）

创建脚手架项目。

```bash
# 基础模式
drun init myproject

# 包含 CI 工作流
drun init myproject -ci

# 强制覆盖
drun init myproject -force
```

## drun convert（格式转换）

从其他格式转换为 YAML 测试用例。

### cURL 转换

```bash
# 基本转换
drun convert sample.curl -outfile testcases/from_curl.yaml

# 脱敏敏感头
drun convert sample.curl -outfile test.yaml -redact Authorization,Cookie

# 提取为变量
drun convert sample.curl -outfile test.yaml -placeholders on
```

### Postman 转换

```bash
# 转换 Collection
drun convert collection.json -outfile testcases/from_postman.yaml

# 带环境文件
drun convert collection.json -outfile test.yaml -postman-env environment.json

# 拆分为多文件
drun convert collection.json -output-mode split -suite-out testsuites/suite.yaml
```

### HAR 转换

```bash
# 基本转换
drun convert recording.har -outfile testcases/from_har.yaml

# 脱敏请求头
drun convert recording.har -outfile test.yaml -redact Authorization
```

## drun convert-openapi（OpenAPI 转换）

```bash
# 基本转换
drun convert-openapi openapi.json -outfile testcases/from_openapi.yaml

# 按标签过滤
drun convert-openapi openapi.json -outfile test.yaml -tags users,orders

# 拆分输出
drun convert-openapi openapi.json -output-mode split -placeholders on
```

## drun export curl（导出 cURL）

将测试用例导出为 cURL 命令。

```bash
# 基本导出
drun export curl testcases/test_api.yaml

# 指定步骤
drun export curl test_api.yaml -steps 1,3-5

# 输出到文件
drun export curl test_api.yaml -outfile export.curl

# 添加注释
drun export curl test_api.yaml -comments on

# 脱敏
drun export curl test_api.yaml -redact Authorization
```

## drun fix（自动修复）

```bash
# 修复所有问题
drun fix testcases/

# 仅修复缩进
drun fix testcases/ -mode spacing

# 仅修复 hooks 格式
drun fix testcases/ -mode hooks
```

## drun server（报告服务器）

启动 Web 报告服务器，`s` 是 `server` 的简写。

```bash
# 默认启动
drun server

# 指定端口
drun server -port 8080

# 不自动打开浏览器
drun server -headless

# 开发模式（自动重载）
drun server -reload
```

功能：
- 自动扫描 `reports/` 目录
- SQLite 报告索引
- 分页列表（每页 18 条）
- 详情查看
- RESTful API: `/api/reports`
