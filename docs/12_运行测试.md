# 运行测试

使用 `drun run`（或简写 `drun r`）命令执行测试。

## 基本用法

```bash
# 运行单个测试文件
drun run testcases/test_login.yaml -env dev

# 省略 .yaml 后缀
drun run test_login -env dev

# 运行目录下所有测试
drun run testcases -env dev

# 运行测试套件
drun run testsuite_smoke -env dev
```

## CLI 按文件内 case 名称精确/多选运行

`drun run` 支持在 `path` 里直接带 case 选择器：

```bash
# 单选：只执行 test_channel_token_flow 文件中 config.name=获取渠道 token 的 case
drun run "test_channel_token_flow:获取渠道 token" -env dev

# 多选：逗号分隔多个 case 名称（精确匹配）
drun run "test_channel_token_flow:获取渠道 token,刷新渠道 token" -env dev
```

说明：

- 匹配键是 case 的 `config.name`，精确匹配，不做模糊/正则
- 多选会自动去重，执行顺序按目标文件里的原始 case 顺序
- 如果名称未命中，会在运行前报错，并提示可用 case 名称列表
- `path:case` 写法不支持 case 名本身包含英文逗号；这类场景建议避免逗号命名

## 套件中按 invoke 文件内 case 选择执行

当你运行 `caseflow` 套件时，可以在套件 YAML 里直接配置 `invoke_case_name` / `invoke_case_names`：

```yaml
caseflow:
  - name: 渠道 Token 流程（单选）
    invoke: test_channel_token_flow
    invoke_case_name: 获取渠道 token

  - name: 渠道 Token 流程（多选）
    invoke: test_channel_token_flow
    invoke_case_names:
      - 获取渠道 token
      - 刷新渠道 token
```

说明：

- 这两个字段只对 `invoke` 步骤生效
- 两者互斥，不能同时写
- 不配置时保持旧行为（按兼容逻辑执行默认 case）

## 默认产物行为

- **脚手架项目内运行**：继续默认生成 `logs/`、`reports/`、`snippets/`
- **非脚手架的临时单文件运行**：默认只在当前目录生成一个日志文件，不自动创建 `logs/`、`reports/`、`snippets/`

```bash
# 临时单文件运行：默认只生成 ./test_tts-<timestamp>.log
drun run ./test_tts.yaml -env-file ./demo.env

# 脚手架项目内运行：继续使用 logs/ reports/ snippets/
drun run testcases/test_login.yaml -env dev
```

## 环境指定

`drun run` 现在支持三种环境入口，优先级从低到高如下：

1. 当前工作目录的 `.env`
2. `-env <name>` 对应的命名环境（优先使用 `.env.<name>`，并合并命名环境配置）
3. `-env-file /path/to/.env`

### 命名环境

```bash
# -env dev → 加载 .env.dev
drun run testcases -env dev

# -env uat → 加载 .env.uat
drun run testcases -env uat

# -env prod → 加载 .env.prod
drun run testcases -env prod
```

### 显式环境文件

```bash
drun run testcases -env-file .env.local
drun run tests/test_tts.yaml -env dev -env-file ./demo.env
```

### 单文件默认 .env

如果当前目录存在 `.env`，即使不传 `-env`，也会自动加载：

```bash
drun run tests/test_tts.yaml
```

环境文件示例：

```bash
# .env.dev
BASE_URL=https://dev-api.example.com
API_KEY=dev-key-xxx

# .env
BASE_URL=http://localhost:8080
API_KEY=local-key-xxx

# .env.prod
BASE_URL=https://api.example.com
API_KEY=prod-key-xxx
```

## 标签过滤

使用 `-k` 参数按标签过滤测试用例：

```bash
# 运行包含 smoke 标签的用例
drun run testcases -env dev -k "smoke"

# 布尔表达式
drun run testcases -env dev -k "smoke and api"
drun run testcases -env dev -k "smoke or e2e"
drun run testcases -env dev -k "not slow"
drun run testcases -env dev -k "smoke and not slow"
drun run testcases -env dev -k "(smoke or regression) and not slow"
```

测试用例中的标签定义：

```yaml
config:
  name: 用户登录
  tags: [smoke, auth, critical]
```

## 变量覆盖

使用 `-vars` 参数覆盖变量：

```bash
# 单个变量
drun run test_login -env dev -vars username=admin

# 多个变量
drun run test_login -env dev -vars username=admin -vars password=123456

# 覆盖环境变量
drun run test_login -env dev -vars BASE_URL=http://localhost:8080
```

## 报告生成

```bash
# HTML 报告
drun run testcases -env dev -html reports/report.html

# JSON 报告
drun run testcases -env dev -report reports/results.json

# Allure 报告
drun run testcases -env dev -allure-results allure-results

# 同时生成多种报告
drun run testcases -env dev \
  -html reports/report.html \
  -report reports/results.json \
  -allure-results allure-results
```

- 临时单文件运行默认**不**生成 HTML 报告；需要时请显式传 `-html`
- `-report`、`-html`、`-allure-results` 只要显式指定，就始终按给定路径输出

## 失败处理

### 快速失败

首次失败时停止执行：

```bash
drun run testcases -env dev -failfast
```

### 重试机制

在用例中配置重试：

```yaml
steps:
  - name: 可能失败的请求
    retry: 3              # 最多重试 3 次
    retry_backoff: 0.5    # 重试间隔（指数退避）
    request:
      method: GET
      path: /api/unstable
```

## 日志控制

```bash
# 日志级别
drun run testcases -env dev -log-level DEBUG
drun run testcases -env dev -log-level INFO
drun run testcases -env dev -log-level WARNING

# 日志文件
drun run testcases -env dev -log-file logs/test.log

# 临时单文件运行默认会在当前目录生成一个日志文件
drun run ./test_tts.yaml -env-file ./demo.env

# 显示响应头
drun run testcases -env dev -response-headers
```

## 敏感信息处理

```bash
# 脱敏（默认 CI 环境）
drun run testcases -env dev -secrets mask

# 显示敏感信息（本地调试）
drun run testcases -env dev -secrets plain
```

脱敏效果：

```
[REQUEST] Headers:
  Authorization: Bearer ***
  X-API-Key: ***
```

## 代码片段

运行测试时自动生成可执行代码片段：

```bash
# 脚手架项目内默认生成 curl 和 python 脚本
drun run test_login -env dev

# 临时单文件运行默认不生成 snippets/
drun run ./test_tts.yaml -env-file ./demo.env

# 禁用代码片段
drun run test_login -env dev -snippet off

# 仅生成 curl
drun run test_login -env dev -snippet curl

# 仅生成 python
drun run test_login -env dev -snippet python

# 自定义输出目录
drun run test_login -env dev -snippet-output exports/
```

生成的代码片段：

```bash
# snippets/20250101-120000/step1_login_curl.sh
#!/bin/bash
curl -X POST 'https://api.example.com/api/login' \
  -H 'Content-Type: application/json' \
  -d '{"username": "admin", "password": "123456"}'
```

```python
# snippets/20250101-120000/step1_login_python.py
import httpx

response = httpx.post(
    'https://api.example.com/api/login',
    headers={'Content-Type': 'application/json'},
    json={'username': 'admin', 'password': '123456'}
)
print(response.status_code)
print(response.json())
```

## 消息通知

```bash
# 启用飞书通知
drun run testcases -env dev -notify feishu

# 多渠道通知
drun run testcases -env dev -notify feishu,dingtalk,email

# 仅失败时通知
drun run testcases -env dev -notify feishu -notify-only failed

# 附带 HTML 报告
drun run testcases -env dev -notify email -notify-attach-html
```

通知配置（环境变量）：

```bash
# .env.dev
FEISHU_WEBHOOK=https://open.feishu.cn/open-apis/bot/v2/hook/xxx
DINGTALK_WEBHOOK=https://oapi.dingtalk.com/robot/send?access_token=xxx
SMTP_HOST=smtp.example.com
SMTP_USER=test@example.com
SMTP_PASS=password
MAIL_TO=team@example.com
```

## 完整示例

### 本地开发调试

```bash
drun run test_login -env dev \
  -log-level DEBUG \
  -secrets plain \
  -response-headers
```

### CI/CD 流水线

```bash
drun run testsuites/testsuite_smoke.yaml -env ci \
  -html reports/smoke.html \
  -report reports/smoke.json \
  -secrets mask \
  -failfast \
  -notify feishu \
  -notify-only failed
```

### 回归测试

```bash
drun run testcases -env uat \
  -k "regression and not slow" \
  -html reports/regression.html \
  -allure-results allure-results
```

## 退出码

| 退出码 | 说明 |
|--------|------|
| 0 | 所有测试通过 |
| 1 | 有测试失败 |
| 2 | 配置错误或文件不存在 |

用于 CI/CD 判断：

```bash
drun run testcases -env ci
if [ $? -eq 0 ]; then
  echo "Tests passed"
else
  echo "Tests failed"
  exit 1
fi
```
